Extract and list all compiler warnings from the most recent Xcode build log. This tool parses the build output in derived data to surface warnings about deprecated APIs, type conversions, unused variables, and other issues the Swift or Clang compiler flags during compilation. Addressing these warnings improves code quality and prepares your project for future SDK changes.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `projectPath` | string | No | Current directory | Absolute path to the `.xcworkspace` or `.xcodeproj` file |
| `derivedDataPath` | string | No | Xcode default | Absolute path to the derived data directory containing build logs |

## Usage Examples

### Check warnings for a project using default derived data

> "Show me all build warnings for FitnessTracker"

```json
{
  "projectPath": "/Users/dev/Projects/FitnessTracker/FitnessTracker.xcodeproj"
}
```

### Check warnings with custom derived data

> "List build warnings using my custom derived data path"

```json
{
  "projectPath": "/Users/dev/Projects/FitnessTracker/FitnessTracker.xcodeproj",
  "derivedDataPath": "/tmp/FitnessTracker-DerivedData"
}
```

### Check warnings for a workspace

> "What warnings does my workspace build produce?"

```json
{
  "projectPath": "/Users/dev/Projects/FitnessTracker/FitnessTracker.xcworkspace"
}
```

### Check warnings using all defaults

> "Show me the build warnings for the project in the current directory"

```json
{}
```

## Tips

<Aside type="tip">
  Run [`xcode_build`](/xcode-pilot-mcp/tools/build/xcode-build/) before using this tool. `build_warnings` reads from the most recent build log, so you need a fresh build to get accurate, up-to-date results.
</Aside>

<Aside type="tip">
  Pay special attention to deprecation warnings. These indicate APIs that will be removed in future iOS versions. Fixing them now prevents your app from breaking when you update the deployment target or Xcode version.
</Aside>

<Aside type="note">
  If you use a custom `derivedDataPath` with [`xcode_build`](/xcode-pilot-mcp/tools/build/xcode-build/), pass the same path here so the tool knows where to find the build logs.
</Aside>
