Run SwiftLint in auto-correction mode to automatically fix violations that have safe, mechanical corrections. This tool modifies your source files in place, applying fixes for issues like trailing whitespace, missing colons, vertical whitespace, and other formatting rules. Violations that cannot be safely auto-corrected are reported but left unchanged.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `path` | string | No | Current directory | Absolute path to the file or directory to fix |
| `config` | string | No | Auto-detected `.swiftlint.yml` | Absolute path to a custom SwiftLint configuration file |

## Usage Examples

### Fix the entire project

> "Auto-fix all SwiftLint violations in my FitnessTracker project"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker"
}
```

### Fix a specific file

> "Fix lint issues in my APIClient file"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker/FitnessTracker/Services/APIClient.swift"
}
```

### Fix with a custom configuration

> "Run SwiftLint auto-fix using the team's config"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker",
  "config": "/Users/dev/Projects/FitnessTracker/.swiftlint.yml"
}
```

## Tips

<Aside type="tip">
  Run [`swiftlint_run`](/xcode-pilot-mcp/tools/quality/swiftlint-run/) first to preview all violations, then use `swiftlint_fix` to auto-correct what it can. Review the remaining violations and fix those manually.
</Aside>

<Aside type="tip">
  Run this tool on your working directory before committing to keep your codebase consistently formatted. It pairs well with a pre-commit hook to enforce style automatically.
</Aside>

<Aside type="caution">
  This tool modifies files in place. Make sure your changes are committed or stashed before running it so you can easily review and revert the auto-corrections if needed using `git diff`.
</Aside>
