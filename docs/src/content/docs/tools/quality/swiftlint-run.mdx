Run SwiftLint on your Swift source files to identify code style violations and potential issues. This tool performs a read-only analysis and reports warnings and errors based on your project's SwiftLint configuration. It does not modify any files -- use [`swiftlint_fix`](/xcode-pilot-mcp/tools/quality/swiftlint-fix/) to auto-correct fixable violations.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `path` | string | No | Current directory | Absolute path to the file or directory to lint |
| `config` | string | No | Auto-detected `.swiftlint.yml` | Absolute path to a custom SwiftLint configuration file |

## Usage Examples

### Lint the entire project

> "Run SwiftLint on my FitnessTracker project"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker"
}
```

### Lint a specific file

> "Check the WorkoutViewModel for lint issues"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker/FitnessTracker/ViewModels/WorkoutViewModel.swift"
}
```

### Lint with a custom configuration

> "Run SwiftLint with our team's strict config on the Sources directory"

```json
{
  "path": "/Users/dev/Projects/FitnessTracker/Sources",
  "config": "/Users/dev/Projects/FitnessTracker/.swiftlint-strict.yml"
}
```

### Lint with default settings

> "Run SwiftLint in the current directory"

```json
{}
```

## Tips

<Aside type="tip">
  Run `swiftlint_run` before committing to catch issues early. Pair it with [`swiftlint_fix`](/xcode-pilot-mcp/tools/quality/swiftlint-fix/) to auto-correct what can be fixed, then manually address the remaining violations.
</Aside>

<Aside type="tip">
  If you have a large project, lint specific directories or files to get faster results. Linting the entire project is thorough but can be slow on codebases with thousands of files.
</Aside>

<Aside type="caution">
  SwiftLint must be installed on your system for this tool to work. Install it via Homebrew with `brew install swiftlint` or include it as a Swift Package Manager plugin in your project.
</Aside>
