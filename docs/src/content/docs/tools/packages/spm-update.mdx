Update all Swift Package Manager dependencies in your project to the latest versions that satisfy the version requirements defined in your `Package.swift` or Xcode project settings. This modifies `Package.resolved` with the new version pins. Use this to pick up bug fixes, security patches, and new features from your dependencies.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `projectPath` | string | No | тАФ | Absolute path to the `.xcodeproj`, `.xcworkspace`, or directory containing `Package.swift`. If omitted, uses the current working directory. |

## Usage Examples

### Update all SPM packages

> "Update all Swift packages to their latest compatible versions"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyApp/MyApp.xcodeproj"
}
```

### Update packages in a Swift library

> "Update dependencies for my Swift package"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyLibrary"
}
```

### Update packages in a workspace

> "Update all SPM dependencies in my workspace"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyApp/MyApp.xcworkspace"
}
```

## Tips

<Aside type="tip">
  After running `spm_update`, always build and run your tests to verify that the updated packages are compatible with your code. Breaking changes can occur even in minor version updates.
</Aside>

<Aside type="tip">
  Commit your updated `Package.resolved` file to source control so that your teammates and CI get the same dependency versions after the update.
</Aside>

<Aside type="caution">
  This updates **all** SPM dependencies at once. If you need to update a single package, adjust the version requirement for that specific package in your `Package.swift` or Xcode project settings, then run `spm_resolve`.
</Aside>
