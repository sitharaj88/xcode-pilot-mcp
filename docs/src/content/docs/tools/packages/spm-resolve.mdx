Resolve and fetch all Swift Package Manager (SPM) dependencies defined in your Xcode project or `Package.swift` file. This downloads the exact package versions specified in `Package.resolved`, ensuring your dependency graph is complete and ready for building. Use this after cloning a repository, switching branches, or when Xcode reports missing packages.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `projectPath` | string | No | ŌĆö | Absolute path to the `.xcodeproj`, `.xcworkspace`, or directory containing `Package.swift`. If omitted, uses the current working directory. |
| `scheme` | string | No | ŌĆö | The Xcode scheme to resolve dependencies for. Useful when a workspace contains multiple projects with different dependency trees. |
| `clonedSourcePackagesDir` | string | No | ŌĆö | Custom directory to store cloned package sources. Useful for CI caches or shared builds. |

## Usage Examples

### Resolve packages for an Xcode project

> "Resolve all Swift packages for my project"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyApp/MyApp.xcodeproj"
}
```

### Resolve packages for a specific scheme

> "Resolve SPM dependencies for the MyApp-Dev scheme"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyApp/MyApp.xcworkspace",
  "scheme": "MyApp-Dev"
}
```

### Resolve with a custom package cache directory

> "Resolve packages and store them in our CI cache folder"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyApp/MyApp.xcodeproj",
  "clonedSourcePackagesDir": "/Users/johndoe/Projects/MyApp/.spm-cache"
}
```

### Resolve a Swift package library

> "Resolve dependencies for my Swift package"

```json
{
  "projectPath": "/Users/johndoe/Projects/MyLibrary"
}
```

## Tips

<Aside type="tip">
  Use `clonedSourcePackagesDir` to point to a persistent cache directory in CI. This avoids re-downloading all packages on every build and can significantly speed up your pipeline.
</Aside>

<Aside type="tip">
  If you are getting package resolution errors after switching branches, try running `spm_resolve` to re-fetch packages matching the `Package.resolved` on the new branch.
</Aside>

<Aside type="note">
  This tool resolves packages to the exact versions in `Package.resolved`. It does not update packages to newer versions. Use `spm_update` for that.
</Aside>
