Stream real-time console output (system logs, `os_log` messages, `print()` statements, and crash information) from a connected physical iOS device. This tool captures the unified log stream from the device for a specified duration, making it invaluable for debugging issues that only reproduce on real hardware such as push notifications, HealthKit access, Bluetooth interactions, and camera usage.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deviceId` | string | Yes | — | The UDID of the target device (obtain from [`physical_device_list`](/xcode-pilot-mcp/tools/device/physical-device-list/)) |
| `timeout` | number | No | `10` | Duration in seconds to capture console output before stopping |

## Usage Examples

### Capture a quick log snapshot

> "Show me the console logs from my iPhone for the last few seconds"

```json
{
  "deviceId": "00008130-001A2B3C4D5E6F78"
}
```

### Extended log capture for debugging

> "Stream logs from my iPhone for 30 seconds while I reproduce the crash"

```json
{
  "deviceId": "00008130-001A2B3C4D5E6F78",
  "timeout": 30
}
```

### Capture logs from a different device

> "Get 15 seconds of console output from the iPad"

```json
{
  "deviceId": "00008103-000A1B2C3D4E5F60",
  "timeout": 15
}
```

### Brief log check after app launch

> "Capture 5 seconds of logs right after launching the app"

```json
{
  "deviceId": "00008130-001A2B3C4D5E6F78",
  "timeout": 5
}
```

## Tips

<Aside type="tip">
  Launch the app with [`physical_device_launch`](/xcode-pilot-mcp/tools/device/physical-device-launch/) right before starting the console capture to see the full startup sequence, including initialization logs, network requests, and data loading.
</Aside>

<Aside type="tip">
  Increase the `timeout` to 30-60 seconds when reproducing intermittent bugs. This gives you a longer window to trigger the issue and capture the surrounding log context. Keep in mind that longer captures produce more output.
</Aside>

<Aside type="caution">
  The console stream includes logs from all processes on the device, not just your app. In a busy capture, look for your app's process name or bundle identifier to filter relevant entries.
</Aside>

<Aside type="note">
  Console output only includes messages logged via `os_log`, `NSLog`, `Logger`, or `print()`. If you need to debug issues that do not produce log output, consider adding temporary logging statements to your code and rebuilding.
</Aside>
