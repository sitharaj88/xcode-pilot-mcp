Analyze an IPA archive to extract comprehensive metadata including the bundle identifier, version numbers, supported architectures, embedded frameworks, entitlements, and minimum deployment target. This tool inspects the IPA without installing it, making it ideal for pre-release validation or auditing third-party builds.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `ipaPath` | string | Yes | — | Absolute path to the `.ipa` file to analyze |

## Usage Examples

### Analyze a release build

> "Analyze the IPA I just exported for FitnessTracker"

```json
{
  "ipaPath": "/Users/dev/Projects/FitnessTracker/build/FitnessTracker.ipa"
}
```

### Inspect a downloaded IPA

> "Check the metadata of this IPA from our CI pipeline"

```json
{
  "ipaPath": "/Users/dev/Downloads/FitnessTracker-v2.3.1-build47.ipa"
}
```

### Analyze an ad-hoc distribution build

> "Analyze the ad-hoc build before sending it to testers"

```json
{
  "ipaPath": "/Users/dev/Archives/FitnessTracker-adhoc-2025-01-15.ipa"
}
```

## Tips

<Aside type="tip">
  Run this tool on every IPA before submitting to App Store Connect. It gives you a quick sanity check on the bundle ID, version numbers, and entitlements without opening Xcode.
</Aside>

<Aside type="tip">
  Compare the output across builds to catch unexpected size increases or missing frameworks. If the total IPA size jumps significantly, check the embedded frameworks section for newly added dependencies.
</Aside>

<Aside type="note">
  The IPA file is a ZIP archive internally. This tool extracts it to a temporary location for inspection and cleans up afterward.
</Aside>
