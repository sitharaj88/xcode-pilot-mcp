Retrieves the absolute filesystem path to an application's container directory on a simulator device. You can request the app bundle container, the data container (where documents, caches, and preferences are stored), or a specific App Group shared container. This is essential for inspecting app data, debugging file storage issues, or verifying Core Data databases.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deviceId` | string | Yes | — | The UDID of the target simulator device. |
| `bundleId` | string | Yes | — | The bundle identifier of the installed app. |
| `container` | string | No | `"app"` | The container type to retrieve. Accepted values: `"app"` (the app bundle), `"data"` (the data/sandbox directory), `"groups"` (list all App Group containers), or a specific App Group identifier (e.g., `"group.com.example.shared"`). |

## Usage Examples

### Get the app bundle container
> "Where is the WeatherApp bundle installed on the simulator?"
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789",
  "bundleId": "com.example.weatherapp"
}
```

### Get the data container
> "Show me the data directory for the WeatherApp so I can inspect its database."
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789",
  "bundleId": "com.example.weatherapp",
  "container": "data"
}
```

### Get a specific App Group container
> "Find the shared container used by the app and its widget extension."
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789",
  "bundleId": "com.example.weatherapp",
  "container": "group.com.example.weatherapp.shared"
}
```

### List all App Group containers
> "Show me all the App Group containers for this app."
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789",
  "bundleId": "com.example.weatherapp",
  "container": "groups"
}
```

## Example Output


## Tips

<Aside type="tip">
  The **data** container is where you will find the app's `Documents/`, `Library/`, `Caches/`, and `tmp/` directories. This is the most useful container type for debugging persistent storage, Core Data SQLite files, or UserDefaults plists.
</Aside>

<Aside type="tip">
  Use the **App Group** container path when debugging data sharing between your main app and extensions (widgets, share extensions, etc.). Pass the group identifier directly, such as `"group.com.example.weatherapp.shared"`.
</Aside>

<Aside type="tip">
  Container paths change each time you reinstall the app. Always fetch the path fresh with this tool rather than caching it between install cycles.
</Aside>
