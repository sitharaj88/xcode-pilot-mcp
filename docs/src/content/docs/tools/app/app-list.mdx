Lists all user-installed applications on a simulator device, returning their bundle identifiers, display names, version numbers, and bundle paths. This is useful for verifying installations, finding the correct bundle ID for other app lifecycle operations, or auditing which apps are present on a test simulator.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deviceId` | string | Yes | — | The UDID of the target simulator device. |

## Usage Examples

### List all installed apps
> "What apps are installed on my test simulator?"
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789"
}
```

### Verify a specific simulator's apps
> "Show me all apps on the iPad Air simulator."
```json
{
  "deviceId": "B9C8D7E6-F5A4-3210-9876-FEDCBA987654"
}
```

## Tips

<Aside type="tip">
  This tool only lists **user-installed** apps, not the built-in system apps (Settings, Safari, etc.) that come with the simulator runtime.
</Aside>

<Aside type="tip">
  Use this tool to quickly look up the exact bundle identifier before calling [`app_launch`](/xcode-pilot-mcp/tools/app/app-launch/), [`app_terminate`](/xcode-pilot-mcp/tools/app/app-terminate/), or [`app_uninstall`](/xcode-pilot-mcp/tools/app/app-uninstall/). Bundle IDs are case-sensitive.
</Aside>

<Aside type="tip">
  If you expect an app to appear but it does not, make sure the simulator is booted and the most recent build was compiled for the correct simulator architecture (`iphonesimulator` destination).
</Aside>
