Removes an installed application from a simulator device using its bundle identifier. This completely deletes the app binary and all associated data containers (documents, caches, preferences) from the simulator.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deviceId` | string | Yes | — | The UDID of the target simulator device. |
| `bundleId` | string | Yes | — | The bundle identifier of the app to uninstall (e.g., `com.example.myapp`). |

## Usage Examples

### Uninstall an app by bundle ID
> "Remove the WeatherApp from my simulator."
```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789",
  "bundleId": "com.example.weatherapp"
}
```

### Clean up a test app after QA
> "Uninstall the staging build of our shopping app from the test device."
```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-1234567890EF",
  "bundleId": "com.example.shopapp.staging"
}
```

## Example Output

```
App uninstalled successfully.
Device: F3A1B2C3-D4E5-6789-ABCD-EF0123456789
Bundle ID: com.example.weatherapp
```

## Tips

<Aside type="tip">
  If the app is currently running, it will be terminated automatically before uninstallation. However, it is good practice to call [`app_terminate`](/xcode-pilot-mcp/tools/app/app-terminate/) first for a clean shutdown.
</Aside>

<Aside type="tip">
  Uninstalling removes **all** app data including documents, caches, and UserDefaults. If you need to inspect the data before removing, use [`app_get_container`](/xcode-pilot-mcp/tools/app/app-get-container/) to locate the data directory first.
</Aside>

<Aside type="tip">
  Not sure of the bundle ID? Use [`app_list`](/xcode-pilot-mcp/tools/app/app-list/) to see all installed apps and their bundle identifiers.
</Aside>
