List all keychains currently in the macOS keychain search list. This shows the path and lock status of each keychain, which is helpful for verifying that certificates are accessible during code signing and for troubleshooting CI/CD environments where keychains may need to be explicitly added or unlocked.

## Tips

<Aside type="tip">
  In CI/CD environments, you may need to create a temporary keychain, import your certificates, and add it to the search list before building. Verify with `keychain_list` that the keychain appears and is unlocked.
</Aside>

<Aside type="tip">
  If a keychain is listed as **Locked**, Xcode cannot access the certificates inside it for code signing. Unlock it via `security unlock-keychain` before building.
</Aside>

<Aside type="note">
  The default login keychain is automatically unlocked when you log into macOS. On headless CI machines, you typically need to unlock keychains manually as part of your build script.
</Aside>
