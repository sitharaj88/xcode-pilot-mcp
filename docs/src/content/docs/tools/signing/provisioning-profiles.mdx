List all provisioning profiles currently installed in `~/Library/MobileDevice/Provisioning Profiles/`. This displays each profile's name, UUID, app ID, team, expiration date, and type (development, ad hoc, App Store, or enterprise). Use this to verify that the correct profiles are available before building or to troubleshoot signing issues.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| — | — | — | — | This tool takes no parameters. |

## Tips

<Aside type="tip">
  If a profile is missing, open Xcode and go to Settings > Accounts > Manage Certificates, then download profiles. Alternatively, download them manually from the Apple Developer portal and double-click to install.
</Aside>

<Aside type="tip">
  Use the **UUID** from this list to match against the `PROVISIONING_PROFILE_SPECIFIER` build setting in your Xcode project if you are configuring manual signing.
</Aside>

<Aside type="note">
  Expired profiles cannot be used for signing. Remove expired profiles and download updated ones from the Apple Developer portal or let Xcode manage them automatically.
</Aside>
