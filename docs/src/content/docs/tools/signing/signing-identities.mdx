List all code signing identities (developer certificates) available in your macOS keychains. This shows both iOS and macOS development and distribution certificates, including their validity status, expiration dates, and associated team identifiers. Use this to verify that the correct signing certificates are installed before building or archiving your project.

## Tips

<Aside type="tip">
  If a certificate shows as expired, open Keychain Access and delete it, then download a new one from the Apple Developer portal or regenerate it in Xcode under Settings > Accounts.
</Aside>

<Aside type="tip">
  Look for the **Team ID** (e.g., `ABCDE12345`) in the output to confirm you are using the correct Apple Developer account, especially if you belong to multiple teams.
</Aside>

<Aside type="note">
  This tool reads from all keychains in your search list. If a certificate is in a keychain that is locked, you may be prompted to unlock it.
</Aside>
