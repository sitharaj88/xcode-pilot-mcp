Shut down a specific running simulator or shut down all running simulators at once. This is useful for freeing system resources, resetting state between test runs, or cleaning up after a testing session.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `deviceId` | string | No | `"all"` | The UDID of a specific simulator to shut down, or `"all"` to shut down every running simulator. |

## Usage Examples

### Shut down all running simulators

> "Shut down all my simulators"

```json
{}
```

### Shut down all simulators explicitly

> "Stop every running simulator"

```json
{
  "deviceId": "all"
}
```

### Shut down a specific simulator by UDID

> "Shut down just the iPhone 16 Pro simulator"

```json
{
  "deviceId": "F3A1B2C3-D4E5-6789-ABCD-EF0123456789"
}
```

## Tips

<Aside type="tip">
  When no `deviceId` is provided, the tool defaults to `"all"`, shutting down every running simulator. This is a fast way to clean up after a testing session.
</Aside>

<Aside type="note">
  Shutting down a simulator will terminate any apps running on it. Make sure to collect any logs or screenshots you need before calling shutdown.
</Aside>

<Aside type="tip">
  Running multiple simulators simultaneously consumes significant CPU and memory. Shut down simulators you are not actively using to keep your Mac responsive.
</Aside>
