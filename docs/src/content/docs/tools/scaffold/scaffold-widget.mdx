Generate a WidgetKit extension target with all required boilerplate including a `TimelineProvider`, `TimelineEntry`, widget view, and configuration. This tool creates either a static or configurable widget, setting up the complete file structure needed to display a widget on the home screen.

## Parameters

| Parameter | Type | Required | Default | Description |
|-----------|------|----------|---------|-------------|
| `name` | string | Yes | — | The name of the widget (e.g., `"DailyStats"` generates a `DailyStatsWidget` extension) |
| `outputPath` | string | Yes | — | Absolute path to the directory where the widget extension folder will be created |
| `kind` | string | No | `"static"` | The widget configuration type. Accepted values: `"static"`, `"configurable"` |

## Usage Examples

### Create a static widget

> "Create a widget called DailyStats for showing fitness progress on the home screen"

```json
{
  "name": "DailyStats",
  "outputPath": "/Users/dev/Projects/FitnessTracker"
}
```

### Create a configurable widget

> "Scaffold a configurable widget called WorkoutGoal that lets users pick which goal to track"

```json
{
  "name": "WorkoutGoal",
  "outputPath": "/Users/dev/Projects/FitnessTracker",
  "kind": "configurable"
}
```

### Create a static widget in a specific extensions folder

> "Generate a static CalorieCounter widget inside the Extensions directory"

```json
{
  "name": "CalorieCounter",
  "outputPath": "/Users/dev/Projects/FitnessTracker/Extensions",
  "kind": "static"
}
```

## Tips

<Aside type="tip">
  Use `"static"` for widgets that display data your app controls (e.g., daily step count). Use `"configurable"` when you want users to customize the widget from the long-press edit screen (e.g., choosing which workout goal to display).
</Aside>

<Aside type="tip">
  After generating the widget, you need to add the new extension target to your Xcode project manually or via Xcode's "Add Target" flow. The generated files provide all the source code, but the target must be registered in the `.xcodeproj` to build.
</Aside>

<Aside type="caution">
  Widgets run in a separate process from your main app. If the widget needs to share data with the host app, you must configure an App Group in both targets and use a shared `UserDefaults` suite or a shared Core Data store.
</Aside>
