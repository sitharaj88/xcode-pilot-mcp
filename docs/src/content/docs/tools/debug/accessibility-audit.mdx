The `accessibility_audit` tool performs an automated accessibility audit on the currently displayed screen of a booted simulator. It inspects the view hierarchy for common accessibility issues such as missing labels, insufficient contrast ratios, unsupported Dynamic Type, small touch targets, and elements hidden from assistive technologies. The audit leverages Xcode's built-in accessibility testing framework, introduced in Xcode 15.

## Parameters

| Parameter  | Type   | Required | Default | Description                                                              |
|------------|--------|----------|---------|--------------------------------------------------------------------------|
| `deviceId` | string | Yes      | —       | UDID of the booted simulator to audit. Must be running iOS 17+ runtime.  |

## Usage Examples

### Audit the current screen

> "Run an accessibility audit on my simulator"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890"
}
```

### Audit after navigating to a specific screen

> "Run an accessibility audit on the settings screen of my app in the simulator"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890"
}
```

### Audit in dark mode

> "Switch to dark mode and run an accessibility audit to check contrast"

First call [`device_appearance`](/xcode-pilot-mcp/tools/debug/device-appearance/) to set dark mode, then run the audit:

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890"
}
```

## Tips

<Aside type="tip">
  Run the audit on every major screen of your app, not just the main view. Accessibility issues often hide in secondary screens like settings, profile editing, or detail views that receive less design attention.
</Aside>

<Aside type="tip">
  Pair the audit with [`device_appearance`](/xcode-pilot-mcp/tools/debug/device-appearance/) to run it in both light and dark mode. Contrast ratio violations frequently appear in only one appearance mode, especially for custom colors that were not tested against both backgrounds.
</Aside>

<Aside type="caution">
  This tool requires **Xcode 15 or later** and a simulator running **iOS 17+**. On older Xcode versions the command will fail. Verify your Xcode version with [`diagnostics`](/xcode-pilot-mcp/tools/debug/diagnostics/) if you encounter compatibility errors.
</Aside>
