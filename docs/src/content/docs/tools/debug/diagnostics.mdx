The `diagnostics` tool gathers a comprehensive diagnostic report from the development environment. The report bundles system information, Xcode version details, simulator states, recent crash logs, system logs, and resource usage metrics into a single output directory. This is invaluable when filing bug reports with Apple, sharing context with teammates, or triaging issues that span multiple subsystems.

## Parameters

| Parameter    | Type   | Required | Default        | Description                                                                                         |
|--------------|--------|----------|----------------|-----------------------------------------------------------------------------------------------------|
| `outputPath` | string | No       | Auto-generated | Directory path where the diagnostic report will be saved. A timestamped directory is created if omitted. |

## Usage Examples

### Collect diagnostics with auto-generated output

> "Run a full diagnostic report"

```json
{}
```

### Save diagnostics to a specific directory

> "Collect a diagnostic report and save it to my project's diagnostics folder"

```json
{
  "outputPath": "/Users/developer/Projects/WeatherApp/diagnostics/2026-02-23-report"
}
```

### Collect diagnostics for a support ticket

> "Generate a diagnostic report on my Desktop so I can attach it to a support ticket"

```json
{
  "outputPath": "/Users/developer/Desktop/xcode-diagnostics-ticket-4521"
}
```

## Tips

<Aside type="tip">
  Run `diagnostics` as the first step when triaging an unfamiliar issue. The report provides a complete snapshot of the development environment, which often reveals version mismatches, missing runtimes, or resource exhaustion problems that are not immediately obvious.
</Aside>

<Aside type="tip">
  When filing a bug report with Apple via Feedback Assistant, attach the full diagnostics directory. It contains exactly the kind of system context Apple engineers need to reproduce and investigate issues.
</Aside>

<Aside type="tip">
  Use a descriptive `outputPath` that includes the date or ticket number (e.g., `/diagnostics/2026-02-23-crash-investigation`). This keeps reports organized and easy to reference later when you have multiple diagnostic snapshots from different debugging sessions.
</Aside>
