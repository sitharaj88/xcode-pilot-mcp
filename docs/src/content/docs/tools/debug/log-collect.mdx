The `log_collect` tool retrieves historical logs from a simulator or physical device for a specified time window. Unlike `log_stream`, which captures logs in real time, `log_collect` reaches back into the device's unified log archive to fetch entries that have already been recorded. This is ideal for post-mortem debugging when you need to examine what happened before, during, or after a crash without having had a live stream running.

## Parameters

| Parameter   | Type   | Required | Default | Description                                                                                       |
|-------------|--------|----------|---------|---------------------------------------------------------------------------------------------------|
| `deviceId`  | string | Yes      | —       | UDID of the simulator or physical device to collect logs from.                                    |
| `predicate` | string | No       | —       | NSPredicate-style filter expression (e.g., `process == "WeatherApp"`).                            |
| `last`      | string | No       | —       | Time range to look back. Accepts durations like `"30s"`, `"5m"`, or `"1h"`.                      |
| `style`     | enum   | No       | —       | Output format. One of `"compact"` (human-readable one-liner per entry) or `"json"` (structured).  |

## Usage Examples

### Collect the last 5 minutes of logs

> "Get me the last 5 minutes of logs from the simulator"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
  "last": "5m"
}
```

### Collect app-specific logs in compact format

> "Show me recent WeatherApp logs from the past minute in compact format"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
  "predicate": "subsystem == \"com.myteam.WeatherApp\"",
  "last": "1m",
  "style": "compact"
}
```

### Collect crash-related logs as JSON

> "Get the last 30 seconds of logs filtered to crash reports, formatted as JSON"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
  "predicate": "eventMessage CONTAINS \"crash\" OR eventMessage CONTAINS \"fatal\"",
  "last": "30s",
  "style": "json"
}
```

### Investigate a networking failure in the last hour

> "Collect all networking logs from the past hour for my app"

```json
{
  "deviceId": "A1B2C3D4-E5F6-7890-ABCD-EF1234567890",
  "predicate": "subsystem == \"com.myteam.WeatherApp\" AND category == \"networking\"",
  "last": "1h",
  "style": "compact"
}
```

## Tips

<Aside type="tip">
  Use `"json"` style when you need to parse log entries programmatically or feed them into another tool. The structured format makes it easy to filter, aggregate, or search through results.
</Aside>

<Aside type="tip">
  Combine `predicate` and `last` to narrow results efficiently. Collecting an hour of unfiltered system logs can produce thousands of entries, so always apply a subsystem or process filter when possible.
</Aside>

<Aside type="tip">
  After a crash, run `log_collect` with `last` set to `"30s"` or `"1m"` and a predicate matching your app's process name. The logs leading up to the crash often reveal the root cause faster than a stack trace alone.
</Aside>
